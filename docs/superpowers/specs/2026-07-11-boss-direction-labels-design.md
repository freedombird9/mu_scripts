# BOSS 同名位置标签设计

## 目标

在 BOSS 刷新浮层中，为同一地图内、同名且坐标不同的两只 BOSS 增加方向标签，帮助快速区分位置。

## 展示规则

- 标签显示在名称最前面，示例：`左 · 傲之煞`、`右 · 傲之煞`。
- 标签使用蓝色粗体；BOSS 名称继续使用原有的高亮颜色和样式。
- 现有地图名与 `(X,Y)` 坐标继续显示，不移除也不改写。
- 仅当同一地图、同名 BOSS 恰好有两个有效且不同的坐标时显示标签。坐标缺失、重合，或同名数量不是两个时不显示，避免错误或含糊的标记。

## 方位计算

对两个坐标计算 X、Y 的绝对差：

- `abs(ΔX) >= abs(ΔY)`：按横向标记。X 小为“左”，X 大为“右”。
- `abs(ΔY) > abs(ΔX)`：按纵向标记。Y 大为“上”，Y 小为“下”。

因此斜向分布也会按偏移更大的轴给出唯一、稳定的标签。

## 实现边界

- 添加一个只读的渲染辅助函数，从已有的 `record` 和 `state.mapBossMarkers` 推导标签。
- 在 `renderRecord` 中，将方向标签和名称分为两个 DOM 节点以分别设置样式。
- 不改动 BOSS 扫描、地图标记读取、记录 ID、合并或持久化逻辑。

## 验证

- 使用固定坐标样例覆盖左/右、上/下、斜向、非双实例和无效坐标的标签推导结果。
- 运行 `node --check` 与 `git diff --check`。
- Chrome 9222 连接恢复可用时，在当前地图确认两个“傲之煞”展示为蓝色粗体的“左”和“右”。
